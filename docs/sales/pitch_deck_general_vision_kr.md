## 슬라이드 2 — 제조업의 큰 문제

### 🎯 공장에서는 매일 사람이 같은 일을 수천 번 반복합니다.

```
검사 작업자:    "이거 불량인가? 멀쩡한가?" — 매일 1,000번
SMT 작업자:    "땜납이 잘 발렸나? 부품 위치 맞나?" — 매일 수천 번
CNC 작업자:    "공구 갈 때 됐나? 속도 맞나? 정밀도 OK?" — 매일 끊임없이
```

→ 사람이 매일 같은 판단을 수천 번 → **피로, 야근, 실수**

**💡 비유**: 도서관 사서가 책 1만 권을 매일 같은 분류로 정리하는 것과 같습니다. AI가 옆에서 도와드리면 사서는 진짜 중요한 판단만 합니다.

---

## 슬라이드 3 — 우리가 도와드리는 3가지

### 🎯 같은 박스가 세 가지 다 합니다. 학습 방식이 같기 때문에.

```
1️⃣  AOI 검사 도우미
     "이 부품 불량인지 멀쩡한지 1차 판단"
     → 사람은 헷갈리는 것만 봄

2️⃣  SMT 생산 도우미
     "땜납·부품·온도가 잘 되어가나 미리 알려줌"
     → 불량 만들기 전에 예방

3️⃣  CNC 가공 도우미
     "공구 마모·진동·온도 보고 자동 조정"
     → 가공 사고·낭비 사전 방지
```

**💡 핵심**: 셋 다 "**사람 옆에서 보고 배우고 도와주는 AI**"라는 점에서 똑같습니다.

---

## 슬라이드 4 — 1️⃣ AOI 검사 도우미

### 🎯 검사기가 "불량" 표시한 100건 중 70건은 헛수고. 그걸 AI가 골라드립니다.

```
검사기:  "100건 불량이에요!"
              ↓
AI 박스: "30건은 진짜 불량, 70건은 멀쩡합니다"
              ↓
작업자:  "고마워. 모르겠는 것만 내가 볼게"
```

**오늘 우리 제품**. 자동차 부품 공장에서 매일 작업자 야근의 원인.

**💡 비유**: 공항 X-ray가 매일 100번 의심 표시했는데, 실제 위험은 30번뿐. 보안요원이 70번 헛고생.

---

## 슬라이드 5 — 2️⃣ SMT 생산 도우미

### 🎯 검사는 "이미 망한 거 분류". 우리는 "망하지 않게 미리 알려줌"으로 확장.

```
[지금까지 — 사후 분류]
공정 → 불량 발생 → 검사 → 폐기
                    ↑
                  여기서만 도움

[확장 — 사전 예방]
공정 → AI가 "땜납 양 줄고 있어요"  → 미리 조정 → 불량 안 만듦
       AI가 "오븐 온도 흔들려요"
       AI가 "부품 위치 비뚤어져요"
```

→ **수율 1% 개선만 해도 라인 1개당 연 수십억 ROI**

**💡 비유**: 의사가 병 걸린 후 치료가 아니라, 미리 건강검진으로 예방하는 것과 같습니다.

---

## 슬라이드 6 — 3️⃣ CNC 가공 도우미

### 🎯 선반·밀링·머시닝센터 — 모든 금속 가공기에 같은 박스 적용.

```
가공 중 박스가 보는 것:
  • 스핀들 부하 (커터에 걸리는 힘)
  • 진동 (chatter)
  • 온도 (열변형)
  • 가공 시간 vs 품질

AI가 자동으로:
  • "공구 마모 임박 → 교체 권장"
  • "지금 속도 줄이면 표면 매끈"
  • "스핀들 열변형 보정 -5µm"
```

→ **공구 비용 30%↓, 가공 시간 10-20%↓, 불량 사전 방지**

**💡 비유**: 자동차에 ABS·차선 이탈 경보가 운전자 도와주는 것과 같습니다. CNC 가공에 운전자 보조 시스템.

---

## 슬라이드 7 — 같은 박스가 셋 다 하는 비결

### 🎯 학습 방식이 같습니다. "사람 결정 → AI 학습 → 점점 자동" 패턴.

```
AOI 검사:    작업자가 "불량/멀쩡" 결정 → AI 학습 → 점점 자동
SMT 생산:    엔지니어가 "온도 +/-" 결정 → AI 학습 → 점점 자동
CNC 가공:    가공사가 "속도 줄여" 결정 → AI 학습 → 점점 자동
                                 ↑
              세 가지 모두 같은 알고리즘
```

→ **연구개발 한 번, 적용은 무한대**.

**💡 비유**: 한국어 잘 배운 사람이 한국어로 요리·운전·강의 다 할 수 있는 것과 같습니다. 같은 언어, 다른 일.

---

## 슬라이드 8 — 박스가 어떻게 학습하나

### 🎯 매일 작업자 결정을 보고 점점 배웁니다. 신입사원처럼.

```
Day 1:     박스 "다 모르겠어요" → 작업자가 100% 결정
Day 30:    박스 "쉬운 건 알겠어요" → 작업자 30%만 결정
Day 180:   박스 "거의 다 알아요" → 작업자 5%만 결정
```

학습 데이터는 **공장 안에서만**. 외부로 안 나감.

**💡 비유**: 좋은 비서가 6개월 후 사장 일정을 알아서 잡는 것과 같습니다. 매일 옆에서 봤기 때문.

---

## 슬라이드 9 — 안전 — 절대 사고 안 나게

### 🎯 박스가 모르겠으면 무조건 사람에게. 사고 0건 수학적 보장.

```
박스 = "모르겠어요"  → 작업자에게 무조건 escalate
                       작업자 = 최종 결정

만약 단 한 번이라도 사고 발생 시
    →  박스 자동으로 보수적 모드 복귀
    →  모든 결정을 다시 작업자에게
```

세 가지 적용 모두 **같은 안전 framework**. 검사 사고 0, 생산 사고 0, 가공 사고 0.

**💡 비유**: 자율주행이 헷갈리면 운전자에게 핸들 넘기는 것과 같습니다. 안전이 최우선.

---

## 슬라이드 10 — 데이터는 공장 안에서만

### 🎯 회사의 비밀(부품 설계·가공 노하우)는 공장 밖으로 안 나갑니다.

```
[공장 내부]
검사 데이터 ─┐
SMT 데이터 ─┼──→ 박스 (공장 안) ──→ 학습 → 더 똑똑해진 박스
CNC 데이터 ─┘
                       │
                       └── 외부로 전송: 0건
```

→ 자동차 부품 비밀·반도체 IP·정밀가공 노하우 안전.

**💡 비유**: 가족 비밀 일기를 집 밖으로 안 가져가는 것과 같습니다. 박스는 공장 안에서만 일합니다.

## 슬라이드 12 — 5년 비전

### 🎯 검사 박스 1개 → 공장 전체를 돕는 AI 플랫폼.

```
2026 (현재)   AOI 검사 도우미                     ← 시작점
                          ↓
2027          + SMT 생산 도우미
                          ↓
2028          + CNC 가공 도우미
                          ↓
2030          제조업 AI 표준 플랫폼               ← 글로벌 표준
                          ↓
2032+         AI 인증 권한 보유 기관              ← 권력
```

같은 박스, 같은 알고리즘, **점점 더 많은 일**.

**💡 비유**: 카카오톡이 메시지 → 결제 → 택시 → 은행 → 게임으로 확장한 것과 같습니다. 한 platform이 점점 커짐.

**💡 비유**: 자동차 안전벨트 의무화처럼, AI도 5년 후엔 인증된 것만 사용 가능. 우리는 그 인증 기관.

## 슬라이드 14 — 실제 검증 결과

### 🎯 검사 도우미 컴퓨터 실험 — 60번 학습에 진짜 불량 놓침 0건.

```
┌──────────────────┬──────────────────┐
│ 놓친 불량         │   작업자 부담     │
│      0건         │     5배 감소      │
│   (60번 학습)    │  (1280 → 230)    │
└──────────────────┴──────────────────┘
```

→ 이건 검사 도우미 결과. **SMT 수율·CNC 가공도 같은 알고리즘이라 비슷한 결과 기대**.

**💡 비유**: 한 분야에서 잘 학습된 AI가 다른 분야에서도 잘 작동하는 게 증명된 셈입니다.
