# aoi-sentinel — 6개월 무료 파일럿

> **귀사의 SMT 라인 가성불량 30%를 6개월 안에 측정 가능하게 줄여드립니다. 도입 비용 0원, 효과 미달 시 전액 무료.**

---

## 한 줄 정의

벤더 무관(Saki / Koh Young / Mycronic / TRI / Mirtec / Omron) AOI 후공정에 USB로 연결되어, 운영자 결정을 학습하면서 가성불량을 자동으로 줄이는 **현장 자가개선 박스**.

## 풀어야 하는 문제

자동차 전장 SMT 라인의 AOI 검사 결과 중 **약 30%가 가성불량(false call)**입니다. 즉 검사기가 NG라 부른 것 중 70% 이상이 사실 OK입니다. 이 차이를 메우는 것은 **사람의 2차 검사** — 노무비, 라인 처리량, 운영자 피로 모두에 매일 비용이 발생합니다.

기존 AI 옵션 (Mycronic DeepReview, Koh Young KSMART AI 등)은 **벤더 종속**입니다. 라인에 검사기 종류가 섞여있으면 모두 따로 도입해야 하고, 데이터가 벤더 시스템 안에 갇힙니다.

## 우리가 다른 점

| | 벤더 솔루션 | aoi-sentinel |
|---|---|---|
| **벤더 종속** | 종속 (해당 검사기만) | **무관** (어떤 벤더든) |
| **데이터 위치** | 벤더 클라우드 | **온프레 only — 공장 외부 안 나감** |
| **개선 방식** | 룰 수동 튜닝 | **운영자 결정으로 자동 학습** |
| **escape 보장** | 없음 | **하드 제약 — 진성불량 누출 시 자동 강등** |
| **확장성** | 검사기 1종 한정 | **2차 검사 → 라인 → 사이트 단위로 확장** |

## 6개월 파일럿 조건

| 항목 | 내용 |
|------|------|
| **도입 비용** | **₩0** (하드웨어·소프트웨어·셋업 무료) |
| **기간** | 6개월, 1라인 |
| **저희 제공** | Edge box 1대 (Jetson Orin Nano), 운영자 UI, 셋업·교육, 24/7 원격 지원 |
| **귀사 제공** | AOI 결과 폴더 read-only 접근, 운영자 1명 일 30분 검토 시간, 6개월 운영 데이터 사용권, 케이스 스터디 동의 |
| **운영 모드** | SHADOW (보조만, 자동 결정 없음) |

**리스크 0**: 박스가 라인 운영에 영향을 주지 않습니다. 기존 워크플로우 그대로, 우리는 운영자 옆에서 보조만 제공합니다.

## 성공 기준 (6개월 시점)

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 가성불량 감소율 | **70% 이상** | 파일럿 시작 시점 30일 vs 종료 시점 30일 직접 비교 |
| escape 건수 | **0건** | 운영자 재검에서 진성불량 누출 0 |
| 운영자 부담 감소 | **월 200시간/라인 이상** | 2차 검사 소요 시간 측정 |

**목표 미달 시**: 자동으로 **계약 종료, 비용 0원**. 박스 회수, 데이터 삭제 옵션 제공.

## 6개월 후 — 본 계약 시 가격

목표 달성 시 다음 중 선택:

| 옵션 | 조건 |
|------|------|
| **A. 영구 라이선스** | 사이트당 ₩100-200M (라인 무제한) + 연 지원 15% |
| **B. 연구독** | 라인당 ₩30-50M/년 (HW 임대 + SW + 지원 일괄) |
| **C. 추가 파일럿** | 다른 라인·다른 벤더 검사기 추가 파일럿 (조건 동일) |

**ROI 참고** (라인 1개 기준):
- 가성불량 70% 감소 → 운영자 야근 ~월 200시간 절감
- 가성불량 재검 비용 ~₩2-5만/건 × 일 200건 절감 = **연 ₩15-35억 절감/라인**
- 본 계약가 ₩100-200M = **2-4개월 회수**

## 왜 지금인가

- 첫 파일럿 3곳까지만 무료 조건 (이후 ₩30-50M 시범가)
- aoi-common-spec 표준 발효 시점 (vendor-neutral 데이터 layer 표준화)
- 자동차 산업 가동률 변동기에 운영비 가시화 필요

## 보유한 기술 신뢰성

- **Mamba 기반 비전 + 강화학습 정책** — Constrained MDP로 escape 제약 보장
- **공개 표준 [aoi-common-spec](https://github.com/DrJinHoChoi/aoi-common-spec)** — 우리가 RFC v0.1을 발행, vendor-neutral 데이터 표준화 주도
- **온프레 only 아키텍처** — 고객 데이터 외부 유출 0 (자동차 QA 인증 친화)
- 오픈소스 reference implementation: [aoi-sentinel](https://github.com/DrJinHoChoi/aoi-sentinel)

## 다음 단계
