## 발표 운영 가이드

### 시간 배분 (15분 기준)

| 단계 | 슬라이드 | 시간 |
|------|---------|------|
| 도입 | 1-3 | 2분 |
| 솔루션 | 4-6 | 3분 |
| 입증 + 차별화 | 7-9 | 3분 |
| **박스 라이브 데모** ⭐ | (별도) | **5분** |
| 제안 + 클로징 | 10-15 | 2분 |

### 슬라이드 + 라이브 데모 결합

슬라이드 7 (학습 곡선) 후에 **박스 데모로 전환**:
1. 책상 위 박스 전원 켜기 → 30초 부팅
2. 화면 + QR 표시
3. 고객 휴대폰으로 QR 스캔 → 같은 UI 확인
4. 가짜 데이터 흘려넣기 → 카드 등장
5. 고객이 직접 PASS/DEFECT 클릭 → KPI 변하는 거 확인
6. 슬라이드 10 (파일럿 조건)으로 복귀

상세: [`docs/demo_playbook_kr.md`](demo_playbook_kr.md)

### 핵심 메시지 3개 (외울 것)

1. **"vendor-neutral + 공개 표준 + 무료 파일럿"** — 경쟁사가 못 흉내냄
2. **"Escape 수학적 보장"** — Lagrangian PPO contract
3. **"6개월 무료, 효과 미달 시 비용 0"** — 의사결정 위험 0
