# aoi-sentinel — 발표자료 (피치 덱)

> **15분 / 15장 / 첫 anchor 미팅 표준.**
> 각 슬라이드는 임원용 큰 메시지(🎯) + 엔지니어용 기술 신뢰(🔧) 동시 제공.

---

## 슬라이드 1 — 표지

### 🎯 AOI 검사 후 매일 작업자가 보는 70% '가짜 불량'을 AI가 알아서 골라드립니다

```
              aoi-sentinel
        — 6개월 무료 파일럿 —
            DrJinHoChoi · 2026
```

**🔧 기술 한 줄**: vendor-neutral edge box (Saki / Koh Young / Mycronic 모두 지원), Mamba RL + Lagrangian PPO 정책, 공개 표준 AICS (BSD-3).

**발표 멘트**: "15장 슬라이드, 15분, 마지막은 한 페이지 LOI입니다."

---

## 슬라이드 2 — 문제

### 🎯 작업자분이 매일 보시는 200건 중 140건은 헛짓입니다

- AOI 검사기 NG 플래그 100건 중
  - **30건 = 진짜 결함** (rework 필요)
  - **70건 = 가성불량** (사실 OK, 시간 낭비)
- 운영자 1-2명이 매일 전담
- 라인 처리량 ↓, 야근 발생, 데이터는 운영자 머릿속 (휘발)

**🔧 기술 한 줄**: 자동차 전장 SMT 라인 평균 false-call rate 30% (산업 표준 측정).

---

## 슬라이드 3 — 기존 솔루션의 한계

### 🎯 다른 회사 솔루션은 '자기네 검사기에서만' 동작합니다

| 옵션 | 한계 |
|------|------|
| Mycronic DeepReview | Mycronic 검사기만 |
| Koh Young KSMART AI | Koh Young 검사기만 + 클라우드 |
| Siemens AOI-FCR | 룰 수동 튜닝, 자동학습 X |

라인에 검사기 두 종류 섞여있으면 → **두 개 따로 도입** + 데이터 vendor에 갇힘.

**🔧 기술 한 줄**: 모든 vendor solution은 closed-source schema, 데이터 휴대 불가, escape rate 보장 없음.

---

## 슬라이드 4 — 우리의 답

### 🎯 어떤 검사기든 OK. 데이터는 공장 밖에 절대 안 나갑니다. 박스는 일하면서 더 똑똑해집니다

- ✅ **벤더 무관** — Saki / Koh Young / Mycronic / TRI 다 지원
- ✅ **온프레 only** — 데이터 외부 export 0
- ✅ **자가학습** — 운영자 클릭 = 학습 라벨
- ✅ **Escape 0건 수학적 보장** — 진짜 결함 놓치는 사고 안 남
- ✅ **공개 산업 표준** — 5년 후에도 데이터 휴대 가능

**🔧 기술 한 줄**: AICS (AOI Common Inspection Schema) 공개 RFC v0.1, github.com/DrJinHoChoi/aoi-common-spec, BSD-3.

## 슬라이드 5 — 어떻게 동작하나

### 🎯 검사기 옆에 USB로 박스 하나 꽂으면 끝

```
[AOI 검사기] → 결과 폴더 → [aoi-sentinel 박스]
                                ↓
                        ┌────────────────┐
                        │ 진짜 NG  → 자동 rework
                        │ 가짜 NG  → 자동 통과
                        │ 모르겠음 → 운영자 호출
                        └────────────────┘
                                ↓
                    운영자 클릭 = 학습 라벨
                                ↓
                       모델 자동 개선
```

라인 운영자가 평소대로 일하면 박스가 알아서 학습.

**🔧 기술 한 줄**: ConvNeXt 이미지 인코더 + Mamba 시퀀스 인코더 + Lagrangian PPO 정책, 3-action {DEFECT, PASS, ESCALATE}.

## 슬라이드 6 — 3단계 점진 도입 (라인 사고 0)

### 🎯 처음 한 달은 박스가 일 안 합니다. 보기만 해요. 그 다음에 천천히 자동.

| 단계 | 기간 | 운영자 부담 | 박스 역할 |
|------|------|-------------|-----------|
| **SHADOW** | Day 1-30 | 100% (평소대로) | 자기 결정 표시만 (영향 0) |
| **ASSIST** | Day 31-180 | 30% → 5%로 점진 감소 | 자신 있는 케이스 자동 결정 |
| **AUTONOMOUS** | Day 181+ (옵션) | KPI 모니터링만 | MES 직접 push |

**Escape 한 건이라도 발생 시 자동으로 보수적 모드로 강등** — 사고 안 남.

**🔧 기술 한 줄**: 모드 전환은 hold-out replay safety gate 통과 시 자동 격상 (`runtime/safety_gate.py`).

## 슬라이드 7 — 입증된 학습 곡선 (VisA 2026-05-04)

### 🎯 4,400장 PCB 데이터로 60번 학습 — 놓친 결함 0건, 비용 5.6배 감소

```
┌──────────────────┬──────────────────┐
│ Escape per iter  │   비용 추세       │
│ 0  0  0  0  0    │  1,280 → 230     │
│ (60 iter 동안 0) │  (5.6× 감소)      │
└──────────────────┴──────────────────┘
┌──────────────────┬──────────────────┐
│ λ (안전 dual)     │  정책 자동 진화   │
│ 자동 조절됨       │  ESCALATE→DEFECT  │
└──────────────────┴──────────────────┘
```

이건 lab 결과 — 실제 [회사명] 라인에선 같은 일이 day 단위로 일어남.

**🔧 기술 한 줄**: NVIDIA A100 80GB, 4,416 VisA images (defect rate 0.091), pure-PyTorch Mamba fallback. 상세 4-차트 plot: [`pilot_evidence_kr.md`](pilot_evidence_kr.md).

## 슬라이드 8 — 우리만 못 따라하는 것

### 🎯 AI 모델은 1-2년이면 카피됩니다. 표준은 카피 못 합니다.

| 자산 | 18개월 후 카피? |
|------|----------------|
| AI 모델 가중치 | ✅ 가능 |
| 어댑터 코드 | ✅ 가능 (BSD-3 공개) |
| **공개 표준 (AICS) — 우리가 정의** | ❌ **불가능** |
| **다중 vendor 누적 라벨 데이터** | ❌ **불가능** (시간만이 만듦) |

**Bluetooth가 노키아 것이 아니듯, 표준은 우리만의 것이 아닙니다. 하지만 우리가 정의했고 운영합니다.**

**🔧 기술 한 줄**: Buffett 경제적 해자 — 표준 거버넌스 + 데이터 플라이휠 + 다중 vendor 어댑터 ecosystem.

## 슬라이드 11 — 성공 기준 (계약서 명시)

### 🎯 가성불량 70% 안 줄면 박스 회수, 비용 0원

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 가성불량 감소율 | **70% ↑** | 첫 30일 vs 종료 30일 비교 |
| Escape 건수 | **0건** | 운영자 재검 ground truth |
| 운영자 부담 감소 | **월 200시간 ↑** | 2차 검사 시간 측정 |

**미달 시 자동 종료** — 박스 회수, 데이터 파기 옵션, 비용 0.

**🔧 기술 한 줄**: 측정은 동일 라인·동일 제품·동일 기간 비교. 통계적 유의성 95% 신뢰구간.

## 슬라이드 12 — 본 계약 가격 (6개월 후, 협의 가능)

### 🎯 라인당 연 절감 ₩10-25억. 2-4개월 만에 본전.

| 옵션 | 가격 | 비고 |
|------|------|------|
| **A. 영구 라이선스** | 사이트당 ₩100-200M | 라인 무제한, 연 지원료 15% |
| **B. 연구독** | 라인당 ₩30-50M/년 | HW 임대 + SW + 지원 |
| **C. 추가 무료 파일럿** | ₩0 | 다른 라인·다른 vendor |

**ROI 참고** (라인 1개 기준):
- 운영자 200hr/월 절감 + 가성불량 재검 비용 절감
- **연 절감액 ₩10-25억** vs 본 계약가 ₩100-200M
- **회수 기간 2-4개월**

**🔧 기술 한 줄**: ROI 추정은 노무비 ₩2-5만/건, 일 200건 false-call 처리 가정. 사이트별 측정 보정.

## 슬라이드 13 — 두 가지 AI 동시 탑재

### 🎯 박스 안에 '안전한 기본 AI' + '자가학습 고급 AI' 둘 다 있습니다

```

│                                                     │
│   [고급] Mamba RL + Lagrangian PPO                  │
```

**Production은 신뢰성, R&D는 차별화 — 둘 다 우리가 갖고 있습니다.**

**🔧 기술 한 줄**: 둘 다 같은 어댑터 SDK + 같은 AICS 스키마 위에서 동작. 모드 전환은 config 한 줄.
